## Technical Explanation

### Why This Works

Rainbow tables store chains as `(start_point, end_point)` pairs. When a password appears at position k in a chain:

1. **Hash the password** → produces a hash value
2. **Walk forward** from that hash for (chain_length - k) steps → produces the chain's endpoint
3. **Search the table** for that endpoint
4. **If found**, load the chain and verify

The attack doesn't care whether the password was:
- In the original wordset (position k=0)
- Generated by reduction (position k>0)

As long as it appears **somewhere** in a stored chain, it can be cracked.
